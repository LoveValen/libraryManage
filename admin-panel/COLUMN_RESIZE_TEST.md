## 📋 测试步骤

### 1. 启动项目

```bash
# 前端
cd admin-panel
npm run dev

# 访问
http://localhost:8081/examples/virtual-table
```

### 2. 基础功能测试

#### 测试单列拖拽
1. 将鼠标悬停在列边界，观察拖拽手柄是否出现
2. 拖拽列边界，验证列宽实时变化
3. 释放鼠标，确认列宽保持

**预期结果：**
- ✅ 拖拽手柄清晰可见（蓝色指示器）
- ✅ 拖拽过程流畅无卡顿
- ✅ 列宽变化实时反映

#### 测试双击自适应
1. 双击任意列边界
2. 观察列宽自动调整

**预期结果：**
- ✅ 列宽自动调整至内容最适宽度

### 3. 高级功能测试

#### 测试多列选择和拖拽
1. 按住 Ctrl 点击多个列标题
2. 拖拽任意选中列的边界
3. 观察所有选中列是否同步调整

**预期结果：**
- ✅ 选中列显示蓝色背景
- ✅ 所有选中列同步调整宽度

### 4. 调试工具测试

#### 开启调试模式
1. 点击"调试模式"复选框
2. 观察拖拽手柄变为红色
3. 查看列宽状态面板

#### 测试拖拽手柄
1. 点击"测试手柄"按钮
2. 观察控制台输出
3. 手柄会短暂高亮显示（红色）

#### 查看列宽状态
1. 点击"查看状态"按钮
2. 打开浏览器控制台查看详细信息

### 5. 性能测试

#### 大数据量测试
1. 设置数据量为 5000
2. 点击"重载"按钮
3. 测试拖拽功能性能

**预期结果：**
- ✅ 虚拟滚动保持 60fps
- ✅ 拖拽延迟 < 16ms
- ✅ 无明显卡顿

## 🛠️ 技术实现细节

### 核心组件
1. **ProTable.tsx** - 主表格组件，支持虚拟滚动和列宽调整
2. **useColumnResize.ts** - 列宽拖拽组合式函数
3. **VirtualTableDemo.vue** - 完整示例实现

### 关键技术
- ElTableV2 虚拟滚动
- 原生拖拽事件处理
- 响应式状态管理
- 性能优化（防抖、节流）

### 样式文件
- `protable.scss` - ProTable 基础样式
- `column-resize.scss` - 列宽拖拽专用样式
