# 🔔 通知系统 API

企业级实时通知和消息系统API，支持多渠道通知、WebSocket实时通信、消息模板管理等功能。

## 🌐 基础信息

**基础路径**: `/notifications`  
**权限要求**: 基于通知类型的动态权限  
**认证方式**: Bearer Token (JWT)  
**实时通信**: WebSocket支持

## 🎯 核心功能

### 📨 通知管理

#### 获取用户通知列表
```http
GET /notifications
```

**权限**: 已认证用户（只能查看自己的通知）

**查询参数**:
```
status: 通知状态 (unread/read/all)
type: 通知类型 (system/borrow/review/security/maintenance)
priority: 优先级 (low/medium/high/urgent)
startDate: 开始日期
endDate: 结束日期
page: 页码 (默认: 1)
limit: 每页数量 (默认: 20, 最大: 100)
```

#### 标记通知为已读
```http
PUT /notifications/{notificationId}/read
```

**路径参数**:
- `notificationId`: 通知ID

**响应示例**:
```json

  "success": true,
  "message": "通知已标记为已读",

    "notificationId": "notif_12345",
```

### 🔄 WebSocket 实时通信

#### 连接WebSocket
```javascript
// WebSocket连接端点
const ws = new WebSocket('ws://localhost:5000/notifications?token=YOUR_JWT_TOKEN');

// 连接建立
ws.onopen = function(event) {
  console.log('WebSocket连接已建立');
};

// 接收消息
ws.onmessage = function(event) {
  const data = JSON.parse(event.data);
  console.log('收到实时通知:', data);
};

// 连接关闭
ws.onclose = function(event) {
  console.log('WebSocket连接已关闭');
};

// 连接错误
ws.onerror = function(error) {
  console.error('WebSocket错误:', error);
};
```

#### WebSocket消息格式

**服务器推送的通知消息**:
```json
{
  "type": "notification",
  "event": "new_notification",
  "data": {
    "notification": {
      "id": "notif_12345",
      "type": "borrow_reminder",
      "title": "图书即将到期提醒",
      "message": "您借阅的《JavaScript高级程序设计》将于明天到期",
      "priority": "medium",
      "createdAt": "2025-01-12T10:30:00.000Z",
      "actions": [
        {
          "type": "button",
          "label": "立即续借",
          "action": "renew_book"
        }
      ]
    }
  },
  "timestamp": "2025-01-12T10:30:00.000Z"
}
```

**系统状态更新**:
```json
{
  "type": "system",
  "event": "maintenance_start",
  "data": {
    "message": "系统将在5分钟后进入维护模式",
    "estimatedDuration": "2 hours",
    "affectedServices": ["borrowing", "search"]
  },
```

**用户状态更新**:
```json
{

  "event": "profile_updated", 
  "data": {
    "userId": 123,
    "changes": ["email", "preferences"],
    "timestamp": "2025-01-12T10:30:00.000Z"
  }
}
```

#### 客户端发送消息
```javascript
// 标记通知为已读
ws.send(JSON.stringify({
  "type": "notification",
  "action": "mark_read",
  "data": {
    "notificationId": "notif_12345"
  }
}));

// 订阅特定类型的通知
ws.send(JSON.stringify({
  "type": "subscription",
  "action": "subscribe",
  "data": {
    "categories": ["borrow", "system"],
    "priorities": ["high", "urgent"]
  }
}));

// 心跳检测
ws.send(JSON.stringify({
  "type": "ping",
  "timestamp": Date.now()
}));
```

## 🚨 错误处理

### 常见错误码

| 错误码 | HTTP状态 | 描述 | 解决方案 |
|--------|----------|------|----------|
| `NOTIFICATION_NOT_FOUND` | 404 | 通知不存在 | 检查通知ID |
| `TEMPLATE_NOT_FOUND` | 404 | 模板不存在 | 检查模板ID |
| `INVALID_RECIPIENT` | 400 | 无效的接收者 | 检查用户ID或邮箱格式 |
| `TEMPLATE_VALIDATION_ERROR` | 400 | 模板验证失败 | 检查模板格式和变量 |
| `CHANNEL_NOT_SUPPORTED` | 400 | 不支持的通知渠道 | 使用支持的渠道 |
| `RATE_LIMIT_EXCEEDED` | 429 | 发送频率超限 | 降低发送频率 |
| `DELIVERY_FAILED` | 500 | 通知发送失败 | 检查渠道配置 |
| `WEBSOCKET_CONNECTION_FAILED` | 500 | WebSocket连接失败 | 检查认证和网络 |

### 通知投递状态

- `queued` - 已排队等待发送
- `sending` - 正在发送中
- `delivered` - 已成功投递
- `failed` - 投递失败
- `bounced` - 被退回
- `unsubscribed` - 用户已取消订阅

## 📊 性能和限制

### 发送限制
- **普通用户**: 无发送权限
- **管理员**: 1000条/小时
- **系统自动**: 无限制
- **批量发送**: 10000条/次

### WebSocket连接
- **最大连接数**: 1000个并发连接
- **心跳间隔**: 30秒
- **重连间隔**: 指数退避（1s, 2s, 4s, 8s...）
- **最大重连次数**: 10次

### 数据保留
- **已读通知**: 90天
- **未读通知**: 1年
- **发送日志**: 6个月
- **统计数据**: 2年
